# Agentic Email — Property Content Pipeline

Retrieve customer and property data using natural language, then generate personalized property highlights powered by AI.

## Overview

This module provides two core capabilities:

1. **Audience & Property Query** — Query a Databricks Genie Space using plain English. Genie translates the question into SQL and returns a customer-property dataset.
2. **Property Highlight Generation** — Generate compelling, personalized property highlights from the query results using Azure OpenAI.

## Setup

### Prerequisites

- Python 3.8+
- Access to a Databricks workspace with a Genie Space configured
- Azure OpenAI deployment

### Install dependencies

```bash
pip install requests pandas python-dotenv openai
```

### Configure environment variables

Create a `.env` file in this directory:

```
DATABRICKS_INSTANCE=your-workspace.azuredatabricks.net
USER_AUTHENTICATION_TOKEN=your-token
GENIE_SPACE_ID=your-genie-space-id
AZURE_OPENAI_ENDPOINT=https://your-resource.openai.azure.com/
AZURE_OPENAI_API_KEY=your-api-key
AZURE_OPENAI_DEPLOYMENT=your-deployment-name
```

## Usage

### 1. Query customer-property data

```python
from genie_audience_property_query import build_query_prompt, genie_audience_property_query

question = "Find customers who have viewed at least 3 listings in Texas in the last 30 days"
prompt = build_query_prompt(question)
result = genie_audience_property_query(prompt)

df = result["result_table"]         # pandas DataFrame
records = result["result_json"]     # list of dicts
query = result["query"]             # SQL query generated by Genie
conversation_id = result["conversation_id"]  # reuse for follow-up questions
```

To continue an existing conversation:

```python
result = genie_audience_property_query(prompt, conversation_id=conversation_id)
```

### 2. Generate property highlights

```python
from openai import AzureOpenAI
from property_content_generator import dedupe_properties, build_prompt, write_property_highlight

client = AzureOpenAI(
    azure_endpoint=os.getenv("AZURE_OPENAI_ENDPOINT"),
    api_key=os.getenv("AZURE_OPENAI_API_KEY"),
    api_version="2024-02-01"
)
model = os.getenv("AZURE_OPENAI_DEPLOYMENT")

instruction = "in 2 sentences, focusing on location and price"

highlights = []
for _, row in df_properties.iterrows():
    prompt = build_prompt(row, instruction)
    highlight = write_property_highlight(client, model, prompt)
    highlights.append({"listing_id": row["listing_id"], "highlight": highlight})

df_highlights = pd.DataFrame(highlights)
```

### 3. Join highlights back to customer-property table

```python
cols = list(dict.fromkeys([c for c in exclude_cols + ["listing_id"] if c in df.columns]))
df_result = df[cols].merge(df_highlights, on="listing_id", how="left")
```

## Functions Reference

### `genie_audience_property_query.py`

| Function | Description |
|---|---|
| `build_query_prompt(question)` | Wraps a natural language question with standard column requirements |
| `genie_audience_property_query(prompt, conversation_id)` | Queries Genie Space, polls until complete, returns DataFrame + metadata |

### `property_content_generator.py`

| Function | Description |
|---|---|
| `dedupe_properties(df, exclude_cols, dedupe_keys)` | Deduplicates property rows |
| `build_prompt(property_row, instruction, customer_row, ...)` | Builds LLM prompt for one or multiple properties, with optional customer context |
| `write_property_highlight(client, model, prompt)` | Calls Azure OpenAI and returns the generated highlight |
